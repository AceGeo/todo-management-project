To project αυτό είναι ένα REST API ενέργειες που αφορά την διαχείριση ενός συνόλου εργασιών (todos). Το αρκτικόλεξο "CRUD" αναφέρεται στις τέσσερις βασικές λειτουργίες που σχετίζονται με τη διαχείριση δεδομένων:

Create (Δημιουργία): Επιτρέπει τη δημιουργία νέων εγγραφών.
Read (Ανάγνωση): Επιτρέπει την ανάκτηση δεδομένων.
Update (Ενημέρωση): Επιτρέπει την τροποποίηση υπάρχοντων εγγραφών.
Delete (Διαγραφή): Επιτρέπει τη διαγραφή υπαρχόντων εγγραφών.
Ο TodoController παρέχει τις παρακάτω ενέργειες:

Create: Η μέθοδος addTodo είναι μια POST ενέργεια που επιτρέπει τη δημιουργία νέας εργασίας (todo) στη βάση δεδομένων.
Read: Οι μέθοδοι getTodo και getAllTodos είναι GET ενέργειες που επιτρέπουν την ανάγνωση ενός συγκεκριμένου todo ή όλων των todos αντίστοιχα.
Update: Η μέθοδος updateTodo είναι μια PUT ενέργεια που επιτρέπει την ενημέρωση μιας υπάρχουσας εργασίας με βάση το ID της.
Delete: Η μέθοδος deleteTodo είναι μια DELETE ενέργεια που επιτρέπει τη διαγραφή μιας υπάρχουσας εργασίας με βάση το ID της.
Complete / Incomplete: Οι μέθοδοι completeTodo και incompleteTodo είναι PATCH ενέργειες που επιτρέπουν την σήμανση μιας εργασίας ως ολοκληρωμένη ή μη ολοκληρωμένη.
Έτσι, ο TodoController παρέχει πλήρη λειτουργίες CRUD για τη διαχείριση των εργασιών (todos) μέσω του δικτυακού πρωτοκόλλου REST API.

Έχει δημιουργηθεί μία βάση δεδομένων μέσω Mysql καθώς και η συνδεσιμότητα με το backend κομμάτι! Όπου η βάση ενημερώνεται αυτόματα με τις αλλαγές που έχουν επιβληθεί μέσω του springboot!

Ουσιαστικά γίνεται αναφορά σε todos δεδομένα , δηλαδή σε διαδικασίες ! Το συγκεκριμένο API δίνει την δυνατότητα να δημιουργηθούν διαδικασίες , και εν συνεχεία ό ταυτοποιημένος καθώς και εξουσιοδοτημένος χρήστης να έχει πρόσβαση στις διαδικασίες αυτές και να μπορεί να τις δεί GET, να κάνει UPDATE DELETE καθώς και CREATE νέων διαδικασίων!

Χρησιμοποιείται authentication και authorization μέσω του spring boot security. όπου μέσω αυτού δίνεται το κατάλληλο authorization καθώς και οι παράμετροι για το authentication στους χρήστες!
